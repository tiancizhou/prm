# PRM Three-Role Permission Regression Checklist

**Date:** 2026-03-12

## Scope

当前系统角色模型已收敛为：

- `SUPER_ADMIN`（超级管理员）
- `PROJECT_ADMIN`（项目经理）
- `DEV`（开发）

本清单用于人工回归当前权限体系是否符合预期，覆盖：

- 登录态权限获取
- 左侧导航显示
- 路由访问控制
- 后台管理页按钮显隐
- 权限页分配弹层
- 需求 / Bug 页主操作按钮

---

## 通用回归步骤

每个角色都建议按下面顺序检查：

1. 使用该角色账号登录
2. 检查左侧一级导航是否符合预期
3. 检查 `后台 / 组织 / 项目集` 能否进入
4. 检查无权限模块是否不可见或不可直接访问
5. 检查页面内主按钮是否按权限显隐
6. 检查权限分配变更后重新登录是否即时生效

---

## 角色一：`SUPER_ADMIN`

### 预期模块可见性

- 应看到：
  - `工作台`
  - `项目集`
  - `组织`
  - `后台`
- 若后续开放 `文档`，也应默认可见

### 预期路由访问

- 任意模块与页面都应可直接进入
- 手工输入任意受控 URL 不应被拦截

### 后台管理页预期

- `后台 > 部门`
  - 可新增一行
  - 可保存当前
  - 可删除当前
  - 可保存直属子部门
- `后台 > 用户`
  - 可添加用户
  - 可编辑用户
  - 可启用/禁用用户
  - 可删除用户
- `后台 > 权限`
  - 可新增分组
  - 可编辑分组
  - 可删除空分组
  - 可打开 `按模块分配权限`
  - 可在弹层里修改模块与动作权限

### 项目页预期

- `需求`
  - 可新建需求
  - 可新建子需求
  - 可编辑需求
  - 可批量编辑
  - 可批量指派
- `Bug`
  - 可报 Bug
  - 可指派 Bug
  - 可编辑 Bug
  - 可删除 Bug

---

## 角色二：`PROJECT_ADMIN`

### 中文业务名称

- 项目经理

### 预期模块可见性

- 应看到：
  - `工作台`
  - `项目集`
- 是否看到 `组织 / 后台`，取决于当前给该角色分配的模块权限

### 路由访问回归

- 若未分配 `admin:view`
  - 左侧不应显示 `后台`
  - 直接输入 `/admin/departments` 应被拦截并跳到安全页面
- 若已分配 `admin:view`
  - 可以进入后台域

### 后台页按钮回归

如果给 `PROJECT_ADMIN` 分配了后台模块和相应动作权限：

- `department:create / update / delete`
- `user:create / update / delete`
- `role-group:create / update / delete`
- `permission:assign`

则页面上对应按钮应显示；未分配时应隐藏。

重点检查：

- `后台 > 用户` 顶部 `添加用户`
- 行内 `编辑 / 启用禁用 / 删除`
- `后台 > 权限` 顶部 `新增分组 / 按模块分配权限`

### 项目页动作回归

若项目经理具备以下动作权限：

- `requirement:create`
- `requirement:update`
- `requirement:assign`
- `bug:create`
- `bug:update`
- `bug:delete`
- `bug:assign`

则应看到相应按钮；否则按钮应隐藏。

重点检查：

- `需求页` 顶部 `新建需求`
- 行内 `编辑`
- `批量指派 / 批量编辑`
- `Bug页` 顶部 `报Bug`
- 行内 `指派`
- `Bug详情页` 中的 `编辑 / 删除 / 指派`

---

## 角色三：`DEV`

### 中文业务名称

- 开发

### 预期模块可见性

- 至少应看到：
  - `工作台`
  - `项目集`
- 是否看到 `组织 / 后台`，取决于是否显式分配了对应模块权限

### 预期路由访问

- 未分配的后台或组织模块不应可见
- 手工输入无权限 URL 应被拦截

### 后台管理页预期

- 默认情况下，开发角色通常不应看到后台管理动作
- 若给了 `admin:view` 但没给动作权限：
  - 后台能进
  - 新增 / 编辑 / 删除 / 分配按钮应隐藏

### 需求页预期

- 若只给 `requirement:view`：
  - 能看需求页
  - 不能新建需求
  - 不能看到需要管理权限的按钮
- 若额外给 `requirement:update / requirement:assign`：
  - 对应按钮出现

### Bug页预期

- 若只给 `bug:view`：
  - 能看 Bug 页
  - 不能报 Bug
  - 不能指派 / 删除 / 编辑
- 若额外给 `bug:create / bug:assign / bug:update / bug:delete`：
  - 对应按钮出现

---

## 关键专项检查

### 1. 模块权限变更是否影响导航

- 给某角色取消 `admin:view`
  - 重新登录后，`后台` 应消失
- 给某角色加上 `organization:view`
  - 重新登录后，`组织` 应出现

### 2. 动作权限变更是否影响按钮

- 取消 `user:create`
  - `后台 > 用户` 顶部 `添加用户` 应消失
- 增加 `permission:assign`
  - `后台 > 权限` 顶部 `按模块分配权限` 应出现
- 取消 `bug:assign`
  - `Bug` 列表与详情中的 `指派` 应消失

### 3. 分配弹层联动是否正确

- 勾选动作 → 自动勾选所属模块
- 取消模块 → 其下动作全部取消
- 保存后重新打开弹层 → 勾选状态保持一致

### 4. 角色收敛是否生效

- 权限页中只应出现：
  - `SUPER_ADMIN`
  - `PROJECT_ADMIN`
  - `DEV`
- 用户编辑时角色下拉只应出现这 3 个

---

## 验收完成标准

当以下条件同时满足，可认为当前 3 角色模型权限体系通过回归：

- 角色列表只剩 3 个核心角色
- 模块权限能控制导航和路由
- 动作权限能控制主按钮显隐
- 权限分配弹层能同时配置模块和动作
- `SUPER_ADMIN` 始终全量放行
- `PROJECT_ADMIN / DEV` 的显示与操作随权限配置变化而变化
